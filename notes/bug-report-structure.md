# Bug Report Structure

A bug report helps the team understand, reproduce, and fix a defect.

## Basic Fields
- Summary
- Environment
- Steps to Reproduce
- Expected Result
- Actual Result
- Severity
- Priority

## Purpose of Each Field

### Summary
A short and specific description of the defect.

### Environment
The system, browser, device, or environment where the defect occurred.

### Steps to Reproduce
The exact actions needed to reproduce the defect.

### Expected Result
What the system should have done.

### Actual Result
What actually happened.

### Severity
How serious the defect impact is.

### Priority
How urgently the defect should be fixed.
