# Severity vs Priority

## Severity
Severity means how serious the defect impact is.
It is about how much the bug affects the system, users, business, security, or data.

## Priority
Priority means how urgently the defect should be fixed.
It is about fix order and business urgency.

## Key Difference
- Severity = impact
- Priority = fix order

## Important Reminder
Severity and Priority are not always the same.
A defect can be severe but not urgent, or less severe but still high priority if it affects an upcoming release.
