# API Basics

## What is an API
An API is an interface that allows two systems to communicate.
It can be understood like a waiter in a restaurant who passes requests and results.

## API in Login
In a login feature, the API sends the username and password to the backend system and returns a success or failure result.

## Why QA should learn API
QA should not only say that a login failed.
QA should also think whether the backend API failed or whether the page handled the API result incorrectly.

## UI Testing vs API Testing
- UI testing checks visible page behavior
- API testing checks backend interface behavior and data transfer

## Why API Testing makes QA stronger
A QA who can do API testing can identify problems faster, write more accurate defect reports, and test system behavior without depending only on the UI.
