# Test Levels and Test Types

## Key Difference
- Test level = where we test
- Test type = what we test

## Test Levels
### Component / Unit Testing
Tests a very small part of the software, such as a function or a small logic unit.

### Integration Testing
Tests whether two or more modules work correctly together.

### System Testing
Tests the whole complete system or full workflow.

### Acceptance Testing
Checks whether the system is acceptable to users or business and ready for release.

## Test Types
### Functional Testing
Checks whether a feature works correctly according to requirements.

### Non-functional Testing
Checks how well the system works, such as performance, security, stability, and usability.

### Smoke Testing
A quick check of the most important core functions.

### Regression Testing
Checks whether a change has caused problems in other existing functions.
