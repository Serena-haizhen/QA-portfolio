# Test Case Structure

A basic test case helps testers describe what to test, how to test it, and what result is expected.

## Basic Fields
- ID
- Title
- Preconditions
- Steps
- Expected Result
- Actual Result
- Status

## Purpose of Each Field

### ID
A unique identifier for the test case.

### Title
A short description of what is being tested.

### Preconditions
Conditions that must be true before the test starts.

### Steps
Actions performed by the tester.

### Expected Result
What the system should do.

### Actual Result
What actually happened during testing.

### Status
The final result of the test case, such as Pass or Fail.
