# AppSec Journey

This folder documents my self-learning journey into Application Security, with a focus on Web and API security testing.

## Background

I started as a self-taught QA engineer learning manual testing, API testing, and test automation. As I went deeper into API testing and HTTP fundamentals, I realized that the same skills used to find bugs are also used to find security vulnerabilities — just from a different angle.

This folder is where I document that transition.

## Focus Areas

- **Web Application Security** — OWASP Top 10, Burp Suite, manual testing for vulnerabilities
- **API Security** — authentication, authorization, input validation, business logic flaws
- **Tools** — PortSwigger Web Security Academy, TryHackMe, HackTheBox, Burp Suite

## Long-term Goal

Become an Application Security Engineer / API Security Specialist — combining QA testing mindset with security expertise.

## Structure

- `week-XX-*.md` — weekly learning notes
- `labs/` — write-ups from PortSwigger, TryHackMe, HTB labs
- `reports/` — security test reports on practice APIs and applications
